# 執筆ルール

## A. 事実の収集と検証
- 以下の専門語を記事内で最初に一度だけ平易に説明する：売上高、営業利益、経常利益、四半期純利益、粗利、PBR、PER、ROE、移動平均線。
- Web検索で一次情報を取得し、各ブロック末尾に日付付きのリンクを添えて出典を明記する。
- 数表では次を整理する：
  - 会社全体：売上、粗利、営業利益、経常利益、純利益（前年同期比）。
  - セグメント別：売上、営業利益（前年同期比）。
  - 指標：PBR、PER（予想）、ROE（取得日を明記）。
- 製品と用途：セグメントごとに代表1〜3点を列挙する。
- 検証ログ：参照した資料のページや図版、PDFの場合はスクリーンショット番号を記載する。
- 未確定点：一次情報で裏付けできなかった事項を記す。
- 事実と解釈は厳密に分離し、Aでは解釈を加えない。

## B. 構造の診断と持続性評価
- Aで得た事実をまず3〜5点で要約し、その後に意見・推論へ進む。
- 各セグメントごとに以下を整理する：
  1. 強み（事実→根拠の順で3点）。
  2. 弱み（事実→根拠の順で3点）。
  3. 主要製品の需要ドライバーを簡潔に列挙する。
  4. 一次情報に基づき持続性ランク（High / Mid / Low）を付与する。
- 全社の運転資本（在庫・売掛金・現金）の動きがキャッシュに与える影響を事実→推論の順で短く記述する。
- 評価ゲートとして次に確認すべき指標を3点列挙する。
- 受注残など将来の見通しに直結する指標やキャッシュフローの評価も可能であれば取り上げる。

## C. 指標比較と投資判断
- 必要な用語（PER、PBR、ROE、EV/EBITDA、P/S）を最初に一度だけ簡潔に説明し、その後は略語を使用する。
- 入力パラメータは以下を網羅する：
  - company, ticker, exchange, sector_name。
  - peer_criteria：地域、規模、事業類型。
  - business_stage。
  - accounting_basis。
  - horizon。
- 業種・事業段階に応じて指標セットを選択する：
  - 成熟・製造業系：PER、PBR、ROE、EV/EBITDA、FCF利回り。
  - 成長・赤字/薄利：P/S、粗利率トレンド、Rule of 40、営業CF。
  - 半導体・景気循環：PER（景気局面補正）、EV/EBITDA、在庫日数。
  - 資源・素材：EV/EBITDA、原材料スプレッド、サイクル平均ROCE。
  - 金融：PBR、ROE、純金利マージン/NIM、不良債権比率、ソルベンシー。
  - 規制産業・公益：P/CF、配当性向、規制資産ベース、RAB/allowed return。
  - REIT・不動産：P/NAV、FFO/株、LTV、稼働率。
- 同業ベンチマークはBで抽出した実質同業を母集団とし、可能であれば最新指標を5〜10社集め中央値と四分位（25/75%）を算出する。景気循環が強い業種では直近2〜3年の範囲中央値も併記する。
- 出力①：比較ミニ表（Company / Peer Median / Peer P25 / Peer P75 / 市場指数）とし、取得日と出典を日付付きリンクで記す。
- 出力②：事実→意見の順で3〜5点の位置づけ解釈を述べ、Bで整理した持続性やドライバーと連動させる。
- 出力③：判断ラベルを Cheap / Fair / Expensive × High / Mid / Low のマトリクスで決定し、条件は3点以内にまとめる。
- 出力④：価格の動きを1文で診断したうえで、以下のうち最大2つのプレイブックを選択し具体ルールを記述する：
  - Pullback型（押し目）。
  - Breakout型（高値抜け）。
  - Value Reversion型（割安修正）。
  - Event-Driven型（決算・規制・設備稼働）。
- 出力⑤：監視KPIを3点挙げ、定義と閾値を明記する。Bの評価ゲートから抽出する。
- 出力⑥：ガードレールとして流動性、予定イベント、ポジション上限、ギャップリスクへの対応を簡潔に示す。

## 全体方針
- A・B・Cそれぞれのセクションで「事実」と「意見」を明確に分離する。
- 出典は必ず一次情報を優先する。
- 記述は常に論理の筋道を明らかにし、結論と条件をセットで提示する。
